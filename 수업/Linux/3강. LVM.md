# 🔧 LVM (Logical Volume Manager)

## 📘 개요
**LVM**은 **Logical Volume Manager**의 약자로, 리눅스에서 **디스크 파티션을 유연하게 관리하기 위한 기술**이다.  
기존의 고정된 파티션 구조보다 **확장성, 유연성, 관리 편의성**을 제공한다.

---

## ⚙️ 주요 개념

| 용어 | 설명 |
|------|------|
| **Physical Volume (PV)** | 실제 물리적 저장 장치 (예: `/dev/sda1`) |
| **Volume Group (VG)** | 여러 개의 PV를 하나의 그룹으로 묶은 단위 |
| **Logical Volume (LV)** | VG 위에 생성된 논리적 파티션으로, 실제 파일 시스템이 올라감 |
| **Physical Extent (PE)** | PV를 구성하는 최소 단위 블록 (일반적으로 4MB 등) |

---

## 🧩 구조 개념도

```
[Physical Volume]
├── /dev/sda1
└── /dev/sdb1
↓
[Volume Group: vg_data]
↓
[Logical Volumes]
├── lv_home
└── lv_var
```


---

## 💡 장점

- 🔄 **동적 크기 조절**: LV의 크기를 실시간으로 늘리거나 줄일 수 있음  
- 🧱 **디스크 통합 관리**: 여러 물리 디스크를 하나의 논리 볼륨으로 묶을 수 있음  
- 🧰 **스냅샷 지원**: 특정 시점의 데이터를 백업 가능  
- 🚀 **디스크 추가 용이**: 시스템 중단 없이 새 디스크를 추가 가능  

---

## 🖥️ 기본 명령어 예시

```bash
# 1. PV 생성
pvcreate /dev/sdb1

# 2. VG 생성
vgcreate vg_data /dev/sdb1

# 3. LV 생성 (10GB 크기)
lvcreate -L 10G -n lv_home vg_data

# 4. 파일 시스템 생성 및 마운트
mkfs.ext4 /dev/vg_data/lv_home
mount /dev/vg_data/lv_home /home
```

| 항목    | 설명                      |
| ----- | ----------------------- |
| 목적    | 디스크 관리의 유연성 향상          |
| 구성 요소 | PV → VG → LV            |
| 대표 장점 | 크기 조절, 스냅샷, 다중 디스크 통합   |
| 사용 예  | 서버, 데이터베이스, 대용량 스토리지 환경 |
