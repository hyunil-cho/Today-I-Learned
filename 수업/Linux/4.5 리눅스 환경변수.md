## 🐧 리눅스 환경 변수(Environment Variables) 완벽 가이드

리눅스 시스템에서 **환경 변수(Environment Variables)**는 시스템의 동작 방식과 사용자 세션의 설정을 제어하는 데 사용되는 **동적인 값**을 저장하는 변수입니다. 프로그램이 실행될 때 필요한 설정 정보나 경로 등을 전달하는 핵심적인 역할을 합니다.

---

### 1. 환경 변수란 무엇이며, 쉘 변수와는 어떻게 다른가요?

환경 변수는 **이름(Name)**과 **값(Value)**의 쌍으로 이루어져 있으며, 시스템 전반에 걸쳐 설정을 공유하는 데 사용됩니다.

* **역할:** 시스템의 언어, 시간대, 기본 편집기 등의 정보를 저장하고, 특히 **실행 파일이나 라이브러리를 찾는 경로**를 지정합니다.
* **프로세스 간 정보 공유:** 현재 실행 중인 쉘과 그 쉘에서 실행되는 **모든 하위 프로세스**에 상속되어 참조할 수 있도록 합니다.

| 구분 | 환경 변수 (Environment Variable) | 쉘 변수 (Shell Variable) |
| :--- | :--- | :--- |
| **정의** | `export` 명령을 통해 시스템 환경으로 내보내진 변수 | 현재 실행 중인 **쉘(Shell)** 내부에서만 유효한 변수 |
| **유효 범위** | 현재 쉘과 **모든 하위 프로세스** | 현재 쉘 **내부**에서만 유효 |
| **생성/등록** | `export VAR_NAME="value"` | `VAR_NAME="value"` |

---

### 2. 주요 환경 변수 예시와 활용

리눅스 시스템에서 가장 자주 사용되며 중요한 역할을 하는 환경 변수들입니다.

| 환경 변수 | 설명 | 예시 값 | 활용 예시 |
| :--- | :--- | :--- | :--- |
| **`PATH`** | **실행 파일**을 찾는 디렉토리 경로의 목록입니다. (경로는 `:`로 구분) | `/usr/local/bin:/usr/bin:/bin` | 명령어(예: `ls`, `grep`)를 **어디서든** 전체 경로 없이 실행할 수 있게 합니다. |
| **`HOME`** | 현재 사용자의 **홈 디렉토리** 경로입니다. | `/home/user_name` | `cd ~` 명령 실행 시 이동하는 경로를 결정합니다. |
| **`USER`** | 현재 로그인한 **사용자 이름**입니다. | `user_name` | 쉘 스크립트에서 현재 사용자를 식별할 때 사용됩니다. |
| **`LANG`** | 시스템의 **언어 및 로케일** 설정입니다. | `ko_KR.UTF-8` | 터미널의 출력 언어와 문자 인코딩을 지정합니다. |

---

### 3. 환경 변수 활용 기본 명령어 💻

환경 변수를 확인, 설정, 제거하는 기본적인 명령어입니다.

#### 3.1. 환경 변수 조회

* **특정 변수 값 확인:**
    ```bash
    echo $<변수명>
    # 예시: $PATH 값 확인
    echo $PATH
    ```
* **모든 환경 변수 목록 확인:**
    ```bash
    printenv
    # 또는
    env
    ```

#### 3.2. 환경 변수 설정 (세션 내 임시 설정)

변수명과 값 사이에 **공백이 없어야** 합니다. `export`를 사용해야 하위 프로세스에 상속됩니다.

```bash
# 새로운 환경 변수 설정
export MY_SERVER="192.168.1.10"

# 기존 PATH 변수에 새로운 실행 경로 추가
export PATH=$PATH:/opt/myapp/bin
```
#### 3.3. 환경 변수 제거
```
unset <변수명>
# 예시
unset MY_SERVER
```

### 4. 환경 변수 영구 설정 방법 💾
터미널을 닫았다가 다시 열어도 환경 변수가 유지되도록 설정하려면 쉘 설정 파일에 추가해야 합니다.

사용자 전용 설정: 개인 사용자에게만 적용됩니다. 가장 일반적인 방법입니다.

Bash 쉘 사용자: ~/.bashrc 또는 ~/.profile 파일의 끝에 export 명령을 추가합니다.

Zsh 쉘 사용자: ~/.zshrc 파일의 끝에 export 명령을 추가합니다.

```
# ~/.bashrc 파일에 추가할 내용
export JAVA_HOME="/usr/lib/jvm/java-17-openjdk"
export NODE_ENV="development"

source ~/.bashrc
# 또는
. ~/.bashrc

시스템 전체 설정 (고급): 시스템의 모든 사용자에게 동일하게 적용됩니다. /etc/profile 또는 /etc/environment 파일을 수정합니다. (일반적으로 시스템 관리자가 사용)
```

## 5. 환경 변수 활용 팁: 보안과 유연성

환경 변수를 사용하는 것은 리눅스 시스템 관리와 개발 작업에 있어 다음과 같은 큰 이점을 제공합니다.

| 구분 | 장점  |
| :--- | :--- | 
| 보안 | 데이터베이스 비밀번호나 API 키 같은 민감한 정보를 소스 코드 외부에 환경 변수로 분리하여 관리할 수 있습니다. 이는 특히 컨테이너 환경(Docker, Kubernetes)에서 널리 사용되는 패턴 |
| 유연성 | 개발, 테스트, 운영 등 실행 환경에 따라 프로그램의 동작(예: 로그 레벨, 서버 주소)을 쉽게 변경할 수 있어, 동일한 코드를 다양한 환경에 배포 용이
 |
